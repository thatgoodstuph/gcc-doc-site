Color

The color system defines the palette and semantic color tokens used throughout the GCC Design System.

Overview

The GCC color system provides a comprehensive palette organized into semantic categories: Primary Colors, Secondary Colors, Hover Colors, Notification Colors, Main Gray Colors, Illustration Colors, and Data Visualization Colors. The system also includes Dark Mode variants for many colors to support accessibility and design flexibility.

Usage Guidelines

Do

  • Use semantic color tokens in code (e.g., color.primary, color.success)
  • Maintain sufficient contrast ratios for accessibility (WCAG AA minimum)
  • Use color in combination with other indicators (icons, text, shapes)
  • Reference notification colors with their outline and background variants when appropriate
  • Use gray colors from the Main Gray Colors palette for neutral elements

Don't

  • Hard-code hex values in components—always use design tokens
  • Use color as the only means of conveying information
  • Create custom color variations outside the defined palette
  • Mix color categories arbitrarily—use colors within their intended category
