Typography

The GCC Design System uses two primary typefaces: GT Walsheim Pro for English (EN) content and Noto Sans Japan for Japanese (JP) content. This page documents the typography tokens, scales, and usage guidelines for both typefaces.

Jump to:

  • GT Walsheim Pro (English)
  • Noto Sans Japan (Japanese)

GT Walsheim Pro

GT Walsheim Pro is the primary typeface for English (EN) content in the GCC Design System.

Overview

GT Walsheim Pro is the primary English typography system used across GCC products. It provides a comprehensive type scale from large display sizes (Display 2XL at 72px) to small utility text (Text XS at 12px), with consistent sizing, line heights, and letter spacing that ensure readability and visual hierarchy.

This typeface is used for all English-language content and interfaces, including headings, body text, labels, and UI elements. The system includes both Display styles (for headings and large text) and Text styles (for body content and smaller text).

Usage

When to use

  • All English-language content across GCC products (Amway+, Wellbeing+, account management, checkout flows).
  • Display styles for page titles, section headings, hero text, and prominent UI elements.
  • Text styles for body content, paragraphs, labels, captions, and secondary information.
  • When designing interfaces that require clear visual hierarchy and consistent typography.

When not to use

  • For Japanese (JP) content—use Noto Sans Japan instead.
  • For decorative or marketing-only typography that intentionally breaks the system (only with design approval).
  • For icon fonts or symbol-based typography (use the Icon system instead).

Dos & Don'ts

Do
  • Use GT Walsheim Pro tokens consistently across English content.
  • Maintain proper line heights for optimal readability (as specified in the type scale).
  • Use semantic HTML elements (h1-h6, p) with appropriate typography tokens.
  • Respect letter spacing (tracking) values, especially for Display styles (-2% for Display 2XL through Display MD).
  • Use Regular, Medium, and Bold weights as defined in the system.
Don't
  • Hard-code font sizes or line heights—always use design tokens.
  • Mix GT Walsheim Pro with other typefaces in the same interface.
  • Use more than 3-4 different text sizes on a single page.
  • Create custom font weights outside the defined system (Regular, Medium, Bold).
  • Override tracking values without design approval.

Anatomy

Each typography style in GT Walsheim Pro consists of the following properties:

  • Font family – GT Walsheim Pro (the typeface name).
  • Font size – the base size in pixels (e.g., 72px for Display 2XL, 16px for Text MD).
  • Line height – vertical spacing between lines in pixels (e.g., 90px for Display 2XL, 24px for Text MD).
  • Line height ratio – calculated as line height ÷ font size (e.g., 90px ÷ 72px = 1.25).
  • Tracking (letter spacing) – horizontal spacing between characters, expressed as a percentage (e.g., -2% for Display styles, 0% for Text styles).
  • Font weight – Regular (400), Medium (500), or Bold (700).

Behavior & States

Typography styles are static and do not change based on user interaction. However, font weights can be used to indicate hierarchy and emphasis:

  • Regular (400) – default weight for most content, provides optimal readability.
  • Medium (500) – used for subtle emphasis, subheadings, or medium-priority content.
  • Bold (700) – used for strong emphasis, primary headings, or high-priority content.

Typography should remain accessible across all states. When text appears in interactive elements (buttons, links), ensure sufficient color contrast and maintain readable font sizes even in disabled or hover states.

Variants

The GT Walsheim Pro system includes two main variant categories:

  • Display styles – Large typography for headings and prominent text (Display 2XL, XL, LG, MD, SM, XS).
  • Text styles – Body and smaller text for content and UI elements (Text XL, LG, MD, SM, XS).

All styles support three weight variants: Regular, Medium, and Bold. Display styles (2XL through MD) use -2% tracking for tighter letter spacing, while Display SM, Display XS, and all Text styles use 0% tracking.

Accessibility

  • Maintain minimum font sizes: Text XS (12px) is the smallest size and should be used sparingly for labels and metadata only.
  • Ensure sufficient color contrast: Text must meet WCAG AA contrast ratios (4.5:1 for normal text, 3:1 for large text at 18px+).
  • Respect user font scaling: Typography tokens should scale proportionally when users increase browser or OS font size.
  • Use semantic HTML: Pair typography tokens with appropriate HTML elements (h1-h6 for headings, p for paragraphs).
  • Maintain readable line lengths: Aim for 50-75 characters per line for optimal readability.
  • Ensure adequate line height: Line height ratios (typically 1.25-1.5) provide comfortable reading spacing.

Tokens & Theming

Typography behavior should be driven by design tokens and not hard-coded values. Typography tokens follow this structure:

  • typography.display.2xl, typography.display.xl, typography.display.lg, etc.
  • typography.text.xl, typography.text.lg, typography.text.md, etc.
  • typography.fontFamily.display – "GT Walsheim Pro"
  • typography.fontWeight.regular (400), typography.fontWeight.medium (500), typography.fontWeight.bold (700)

Implement these as CSS variables or theme values so typography can be adjusted centrally. Do not hard-code font sizes, line heights, or letter spacing in individual components.

See Color for text color tokens and Spacing for how typography relates to spacing tokens.

Code & API

Typography tokens can be implemented using CSS variables or theme values. Example:

/* CSS Variables */
:root {
  --font-family-display: "GT Walsheim Pro", sans-serif;
  --font-weight-regular: 400;
  --font-weight-medium: 500;
  --font-weight-bold: 700;
  
  --typography-display-2xl-size: 72px;
  --typography-display-2xl-line-height: 90px;
  --typography-display-2xl-tracking: -2%;
  
  --typography-text-md-size: 16px;
  --typography-text-md-line-height: 24px;
  --typography-text-md-tracking: 0%;
}

/* Usage */
.display-2xl {
  font-family: var(--font-family-display);
  font-size: var(--typography-display-2xl-size);
  line-height: var(--typography-display-2xl-line-height);
  letter-spacing: var(--typography-display-2xl-tracking);
  font-weight: var(--font-weight-regular);
}

.text-md {
  font-family: var(--font-family-display);
  font-size: var(--typography-text-md-size);
  line-height: var(--typography-text-md-line-height);
  letter-spacing: var(--typography-text-md-tracking);
  font-weight: var(--font-weight-regular);
}

Integrate these tokens into your design system theme so that components can consume them without redefining typography values. Always use tokens rather than hard-coded pixel values.

Noto Sans Japan

Noto Sans JP is the primary typeface for Japanese (JP) content in the GCC Design System.

Overview

Noto Sans Japan (JP) is the typography system used for all Japanese-language content and interfaces in GCC products. It provides a comprehensive type scale optimized for Japanese character rendering, with appropriate sizing and leading values that ensure readability for Japanese text.

This typeface is specifically designed for Japanese content and includes additional utility styles not available in the English typeface.

Type Scale

The Noto Sans JP type scale includes display styles, text styles, and utility styles. Each style specifies font size and line height in pixels. All styles support Regular, Medium, and Bold weights.

Display Styles

Token Size / Line Height Line Height Ratio Tracking Weights Usage
Display 2XL 72px / 90px 1.25 -2px Regular, Medium, Bold Large hero displays, major page titles
Display XL 60px / 75px 1.25 -1px Regular, Medium, Bold Large displays, prominent headings
Display LG 48px / 60px 1.25 -1px Regular, Medium, Bold Large headings, section titles
Display MD 36px / 45px 1.25 0% Regular, Medium, Bold Medium headings, subsection titles
Display SM 32px / 40px 1.25 0% Regular, Medium, Bold Small headings, component titles
Display XS 24px / 30px 1.25 0% Regular, Medium, Bold Extra small headings, emphasized text

Text Styles

Token Size / Line Height Line Height Ratio Tracking Weights Usage
Text XL 20px / 26px 1.3 0% Regular, Medium, Bold Extra large body text
Text LG 18px / 25px 1.389 0% Regular, Medium, Bold Large body text, emphasized content
Text MD 16px / 24px 1.5 0% Regular, Medium, Bold Default body text, paragraphs
Text SM 14px / 21px 1.5 0% Regular, Medium, Bold Secondary text, captions, helper text
Text XS 12px / 16px 1.333 0% Regular, Medium, Bold Labels, fine print, metadata

Utility Styles

Token Size / Line Height Line Height Ratio Tracking Weights Usage
Utility XXS 10px / 13px 1.3 0% Regular, Medium, Bold JP-only utility text, very small labels
Utility XXXS 8px / 11px 1.375 0% Regular, Medium, Bold JP-only utility text, smallest labels

Note: Utility styles are JP-only and do not have English equivalents. These are designed specifically for Japanese typography needs.

Japanese-Specific Tokens

Noto Sans JP includes tokens that are unique to Japanese content and do not have English equivalents:

  • Utility XXS (10px / 13px) - JP-only utility text
  • Utility XXXS (8px / 11px) - JP-only utility text, smallest size

These tokens are designed specifically for Japanese typography needs and should be used when appropriate for Japanese content.

Usage Guidelines

Do

  • Use Noto Sans JP tokens consistently across Japanese content
  • Use increased leading (line height) for Japanese text to improve readability
  • Use semantic HTML elements (h1-h6, p) with appropriate typography tokens
  • Reference the mapping table when localizing from English content

Don't

  • Hard-code font sizes or line heights—always use design tokens
  • Mix Noto Sans JP with other typefaces in the same interface
  • Use English typography tokens (GT Walsheim) for Japanese content
  • Use condensed styles for Japanese text (no condensed JP styles available)
  • Create custom font weights outside the defined system
  • Use Utility styles for body text—they are designed for very small labels only
