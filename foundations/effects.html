Effects

Effects in the GCC Design System define how elevation, depth, and overlays are rendered across light and dark modes.

Overview

Effects provide depth, hierarchy, and focus. The primary effect tokens defined in the GCC library are drop shadows for cards and surfaces in both light and dark modes. These tokens ensure consistent elevation behavior across components such as cards, modals, drawers, and dropdowns.

Effects must always be implemented via design tokens (e.g., effect.card.shadow), never via hard-coded shadow values.

Effect Tokens

The following tokens are defined in the Figma library (Effects – Figma).

Card Drop Shadow (Light Mode)

Token Description Box Shadow Color Usage
effect.card.shadow Default card drop shadow in light mode 0px 2px 16px 0px rgba(44,44,44,0.1) #2c2c2c @ 10% opacity Cards, elevated surfaces, dropdowns, and flyouts in light mode

Card Drop Shadow (Dark Mode)

Token Description Box Shadow Color Usage
effect.card.shadow.dark Card drop shadow for dark mode surfaces 0px 2px 16px 6px rgba(0,0,0,0.84) #000000 @ 84% opacity Cards, elevated surfaces, dropdowns, and flyouts in dark mode

Usage Guidelines

Do

  • Use the card shadow tokens for all elevated surfaces (cards, dropdowns, drawers, modals).
  • Keep elevation consistent within a view (e.g., all primary cards use the same shadow token).
  • Pair shadows with appropriate background colors from the Color foundation (e.g., main gray and dark mode gray tokens).

Don't

  • Hard‑code custom box-shadow values in components.
  • Stack multiple different shadow tokens on the same surface.
  • Use heavy shadows on low-contrast backgrounds that reduce readability.

Examples

The following examples illustrate how effect tokens are applied to GCC components:

  • Cards: Product cards and education cards use effect.card.shadow in light mode and effect.card.shadow.dark in dark mode.
  • Modals: Modal containers use the card shadow tokens combined with modal background colors from the Color foundation.
  • Drawers & Slide Overs: Use the same elevation tokens as cards to maintain a consistent depth hierarchy.

Accessibility Considerations

  • Do not rely on shadows alone to convey elevation—use borders, color, and spacing in combination.
  • Ensure that elevated surfaces maintain sufficient contrast against their backgrounds.
  • Verify that focus rings remain visible on elevated surfaces with shadows applied.

For related tokens, see Color (Drop Shadow and Modal Background) and Spacing for layout and spacing scales.
