Components

Reusable UI components that form the building blocks of GCC interfaces.

What are Components?

Components are reusable, self-contained UI elements that implement specific design patterns. Each component follows consistent design principles and can be composed together to build complete interfaces.

Components in this section include:

  • Badge - Small status indicators and labels
  • Button - Interactive buttons for actions
  • Card - Container for related content
  • Drawer - Side panel navigation and content
  • Input Field - Text input and form controls
  • Modal - Dialog overlays for focused interactions
  • Tabs - Tabbed navigation and content organization
  • Table - Data tables for structured information

How to Use Components

For Designers

  • Reference component specifications when designing interfaces
  • Use components consistently across designs
  • Follow component usage guidelines and patterns
  • Check component states and variants before designing

For Developers

  • Import and use components from the design system library
  • Follow component APIs and prop specifications
  • Use design tokens instead of custom styling
  • Maintain accessibility attributes and semantic HTML

Component Pages

Each component page includes:

  • Overview - What the component solves and when to use it
  • Usage - When to use and when not to use
  • Anatomy - Component parts and structure
  • Behavior & States - Interactive states and responsive behavior
  • Variants - Available component variations
  • Accessibility - Accessibility considerations and requirements
